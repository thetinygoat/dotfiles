# GENERAL Guidelines

1. **Do not write or modify code unless explicitly instructed.** Never touch files without the user's permission.
2. **Ask clarifying questions before implementing anything.** Do not assume missing details.
3. **Read relevant files before acting.** Gather context from the codebase rather than guessing.
4. **Review repository documentation first.** Use existing docs to understand expectations and conventions.
5. **Read files before making updates** Don't assume that the last update you made to a file is still in effect, read the file before making any updates to it.

# Tooling Guidelines

1. **Use the tooling already configured in the project.**

- **Node.js:** inspect `package.json` for dependencies, scripts, and tooling.
- **Python:** identify whether the project uses uv, pip or another tool.
- **Other languages:** follow the existing build, formatting, testing, and package tools defined in the repo.
